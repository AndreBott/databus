System Access

Databus provides two distinct ways of accessing its functionality: the Databus User Interface and the Databus API.

The Databus User Interface is a web interface that is mainly used for administering the Databus system components.

The Databus API privides a direct HTTP connection to the underlying Databus system.

While both interfaces provide overlapping functionality, the Databus User Interface should be kept to administration uses whereas the API interface should be used for lower level functionality such as provisioning, uploading and downloading data.

The major difference between accessing the two interfaces is that the Databus User Interface requires full NREL User Authentication (username + password) while the Databus API requires Databus Authentication (username + API Key).

Accessing the Databus API

The Databus API can be accessed with your NREL username and your Databus generated API Key. The Databus generated API Key is created the first time a user logs into the Databus system. The API Key is used as a password of sorts so that your NREL credentials cannot be compromised.

The generated API Key for the current user:

Username API Key
${user.name} ${user.apiKey}
This API Key should be used everywhere a password is requested when accessing the Databus API.

An alternative to accessing the Databus API with your User account is to use a Robot account. These are accounts that have been created by a group administrator for the specific purpose of accessing restricted data but without exposing User credentials. The Robot name and API Key must be known in order to access Databus as a Robot.

The current list of Robots that are available to you are:

#{list user.robots, as:'robot'} #{/list}
Robot Username API Key
${robot.name} ${robot.apiKey}

One limitation of the Robot account is that it cannot log directly into the User Interface nor can it create User Groups or other Robots. In order to perform these actions, you must log in with your User account.
